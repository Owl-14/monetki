# 🪙 Монетки

Внутренний офис команды: клиенты, задачи, финансы и сотрудники в одном месте.

**Сайт:** https://owl-14.github.io/monetki

## Что умеет

- **Два направления** — 🎾 Падел и 💻 Разработка — с изоляцией данных: сотрудник видит только своё направление, админы видят всё.
- **Падел**: база «Площадки» (где проходят турниры) и «Игроки» (с импортом из Excel).
- **Разработка**: клиенты с воронкой (Лид → Переговоры → В работе → Поддержка → Отказ).
- **Задачи**: постановка сотрудникам и себе, сроки, приоритеты, обсуждение внутри задачи, уведомления.
- **Финансы** (только для админов): доходы/расходы по направлениям, вручную + автоматическая выписка из Точка Банка с определением способа оплаты (счёт / карта / СБП / наличные).
- **PWA**: ставится на телефон как приложение, работает офлайн, шлёт уведомления.

## Как устроено

| Часть | Технология | Где |
|---|---|---|
| Интерфейс | Статический сайт (HTML/CSS/JS, без сборки) | GitHub Pages, этот репозиторий |
| База данных | Google Таблица | Google Диск владельца |
| Сервер (права, банк) | Google Apps Script | [`google-apps-script/Code.gs`](google-apps-script/Code.gs) |
| Банк | API Точка Банка (выписка по расписанию) | токен хранится в свойствах скрипта, **не** в репозитории |

Пока общая база не подключена, сайт работает в **демо-режиме** (данные в localStorage браузера).
Коды входа в демо: `111111` — админ, `222222` — сотрудник падела, `333333` — сотрудник разработки.

## Установка общей базы

Пошаговая инструкция — в [SETUP.md](SETUP.md).

## Структура

```
index.html            — оболочка приложения
config.js             — настройки (адрес бэкенда)
css/style.css         — стили
js/app.js             — интерфейс и логика
js/store.js           — слой данных (демо / Google Таблицы)
sw.js                 — service worker (офлайн, уведомления)
manifest.webmanifest  — манифест PWA
google-apps-script/   — серверный код для Google Apps Script
SETUP.md              — инструкция по подключению базы и банка
```
