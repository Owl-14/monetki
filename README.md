# 🪙 Монетки

Внутренний офис команды: клиенты, задачи, финансы, корты и сотрудники в одном месте.

**Сайт:** https://owl-14.github.io/monetki (работает как приложение — ставится на телефон)

## Что умеет

- **Два направления** — 🎾 Падел и 💻 Разработка — с изоляцией: сотрудник видит только своё направление, финансы видят только админы.
- **Падел**: площадки с календарём кортов (будни 18:00–23:00, статусы «забронировано / свободно / занято / хотелось бы» + цена часа), база игроков с импортом из Excel.
- **Разработка**: клиенты с воронкой (Лид → Переговоры → В работе → Поддержка → Отказ).
- **Задачи**: админ ставит любому, сотрудник — только себе; статусы «Не видел» (красный) → «В работе» → «Выполнена» / «Есть вопросы»; обсуждение внутри задачи; уведомления.
- **Финансы** (только админы), вкладки:
  - *Операции* — доходы/расходы по месяцам или за всё время, автоматическая выписка из Точка Банка каждый час, остаток счёта, способ оплаты (счёт/карта/СБП/наличные);
  - *Траты* — траты сотрудников с обязательным фото чека; возврат со счёта или из наличных;
  - *Счета* — личные счета владельцев: (доходы − расходы) направления по долям (Разработка 50/50; Падел 34/33/33), персональные расходы;
  - *Наличные* — кассы Саввы и Андрея, отдельно от общей статистики.
- **Зарплата**: со счёта или наличными, с зачётом непогашенных трат сотрудника; у сотрудника — раздел «Деньги» с историей выплат и трат.
- **Темы**: светлая / тёмная / как в системе.

## Как устроено

| Часть | Технология |
|---|---|
| Интерфейс | статический сайт (HTML/CSS/JS без сборки), PWA, GitHub Pages |
| Сервер | Supabase Edge Function (`supabase/functions/api/index.ts`) |
| База | Supabase Postgres (таблица `records` c JSON-записями), закрыта RLS |
| Банк | API Точка Банка: выписка + баланс, синхронизация по расписанию |
| Деплой | merge в `main` → Pages (фронт) + GitHub Action (бэкенд) автоматически |

Вход — по личному 6-значному коду сотрудника (выдаётся в разделе «Команда»).
Без подключённой базы сайт работает в демо-режиме на localStorage.

## Структура

```
index.html            — оболочка приложения
config.js             — адрес бэкенда
css/style.css         — стили и темы
js/app.js             — интерфейс (роутер, разделы, формы)
js/store.js           — слой данных (демо/сервер), справочники, расчёт счетов
sw.js                 — service worker (кэш, офлайн, уведомления)
supabase/functions/api/index.ts — серверная логика (права, банк, файлы)
supabase/migrations/  — SQL схемы базы
.github/workflows/    — автодеплой бэкенда
google-apps-script/   — архив старого бэкенда
```
