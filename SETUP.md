# Монетки — подключение общей базы и банка

Сайт работает сразу: https://owl-14.github.io/monetki (демо-режим, данные только в вашем браузере).
Чтобы у всей команды была **общая база** в Google Таблицах — выполните шаги ниже. Это копипаст на 10 минут, программировать ничего не нужно.

---

## Шаг 1. Создать «сервер» в Google Apps Script

1. Откройте [script.google.com](https://script.google.com) под своим Google-аккаунтом → **«Новый проект»**.
2. Слева в редакторе будет файл `Код.gs`. Сотрите его содержимое и вставьте **весь** текст из файла [`google-apps-script/Code.gs`](google-apps-script/Code.gs) этого репозитория.
3. Сверху нажмите 💾 (сохранить). Назовите проект, например, «Монетки».
4. В выпадающем списке функций (рядом с кнопками «Выполнить»/«Отладка») выберите **`setup`** → нажмите **«Выполнить»**.
   - Google попросит разрешения — нажмите «Разрешить» (предупреждение «приложение не проверено» → «Дополнительные настройки» → «Перейти»). Это ваш собственный скрипт, он работает только с вашей таблицей.
5. Внизу откройте **«Журнал выполнения»** — там будет:
   - ссылка на созданную таблицу «Монетки — база данных» (она появится и на вашем Google Диске);
   - **КОД ДЛЯ ВХОДА админа** — сохраните его, по нему вы будете заходить на сайт.

## Шаг 2. Опубликовать веб-приложение

1. Справа сверху: **«Начать развёртывание»** (Deploy) → **«Новое развёртывание»**.
2. Тип: **«Веб-приложение»**. Настройки:
   - *Выполнять от имени:* **От моего имени** (ваш аккаунт)
   - *У кого есть доступ:* **У всех** (не переживайте: без кода доступа скрипт никому ничего не отдаёт)
3. Нажмите «Начать развёртывание» и **скопируйте URL** вида `https://script.google.com/macros/s/…/exec`.

## Шаг 3. Подключить сайт к базе

Вариант А (проще): пришлите этот URL мне в чат — я впишу его в `config.js`, и база подключится у всех автоматически.

Вариант Б (самостоятельно): откройте сайт → войдите (пока в демо) → раздел **«Ещё»** → «Общая база» → вставьте URL → «Сохранить». Так база подключится только на этом устройстве, поэтому вариант А лучше.

После подключения зайдите под кодом админа из шага 1, раздел **«Команда»** → добавьте сотрудников. У каждого появится свой 6-значный код входа (🔑 в списке) — отправьте его сотруднику.

## Шаг 4. Точка Банк: автоматическая выписка

Вам понадобится **токен** (длинная строка, JWT-ключ). Где он берётся в Точке: **«Сервисы» → «Все сервисы» → «Интеграции и API» → «Создать JWT-ключ»** (если уже создавали — он в блоке «JWT-ключи» там же). «Айди клиента» (client_id) рядом с ключом пока не нужен — он пригодится позже для мгновенных уведомлений.

Куда вставить токен — дословно:

1. Откройте ваш проект на [script.google.com](https://script.google.com).
2. Слева нажмите ⚙️ **«Настройки проекта»** (Project Settings).
3. Прокрутите вниз до **«Свойства скрипта»** (Script Properties) → **«Добавить свойство скрипта»**.
4. Добавьте две строки (значения вставляете свои, целиком, без кавычек и пробелов):

   | Свойство | Значение |
   |---|---|
   | `TOCHKA_TOKEN` | ваш JWT-ключ (длинная строка) |
   | `TOCHKA_UNIT` | `dev` или `padel` — к какому направлению по умолчанию относить операции банка |

5. Нажмите «Сохранить свойства скрипта».
6. Вернитесь в редактор кода → выберите функцию **`tochkaTest`** → «Выполнить» → в журнале должен появиться список ваших счетов. Если ошибка — пришлите мне текст из журнала (сам токен не присылайте).
7. Выберите функцию **`tochkaSync`** → «Выполнить» — операции за последние 7 дней появятся в разделе «Финансы».
8. Автозагрузка по расписанию: слева ⏰ **«Триггеры»** → «Добавить триггер»:
   - функция: `tochkaSync`
   - источник: «Триггер по времени» → «Таймер по часам» → **«Каждый час»**

> ⚠️ Токен хранится только в свойствах вашего скрипта Google. В публичный репозиторий он не попадает. Никому его не пересылайте, включая меня.

## Шаг 5. Установить как приложение (PWA) и включить уведомления

- **iPhone**: откройте сайт в Safari → кнопка «Поделиться» → **«На экран “Домой”»**. Запускайте с иконки.
- **Android**: Chrome → меню ⋮ → **«Установить приложение»**.
- В приложении: раздел «Ещё» → «Включить уведомления» → «Разрешить».

Уведомления о новых задачах, сообщениях и дедлайнах приходят, когда приложение открыто или свёрнуто. Полноценные push «в любой момент» требуют отдельного push-сервера — это следующий этап.
